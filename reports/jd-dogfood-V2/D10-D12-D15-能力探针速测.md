## D12 暗黑模式 探针 (方法不可行)

**方法**: `vortex_evaluate` 测 `matchMedia().matches` (验证方法不可行)

**实测** (淘宝服饰搜索页, 2026-06-09):
- 系统 dark mode 开启 (date 显示 21:26, 系统主题暗色)
- `vortex_evaluate matchMedia('(prefers-color-scheme: dark)').matches` 返 `true` (MAIN world 下匹配浏览器/系统主题)
- **`matchMedia().matches` 只读, 无法切换 dark**

**结论**:
- **方法不可行**: 切 dark mode 需 CDP `Emulation.setEmulatedMedia`, `vortex_evaluate` 在 MAIN world 触不到 CDP
- **真 gap**: 缺 `vortex_color_scheme` 工具 (CDP `Emulation.setEmulatedMedia` 封装, 支持切 prefers-color-scheme / prefers-reduced-motion 等媒体查询)
- **京东无 dark mode**: 即使加了工具, 京东主站不响应 dark mode, **ROI 低**
- **淘宝部分响应**: 淘宝在某些区块 (如个人中心) 响应 dark, 但搜索/详情页**不响应**

## D15 错误恢复 探针 ("假断网")

**方法**: `vortex_evaluate` 改 `navigator.onLine` (验证"假断网"性质)

**实测** (淘宝服饰搜索页, 2026-06-09):
```js
// 步骤 1: 改 navigator.onLine (JS 读数)
Object.defineProperty(navigator, "onLine", {get: () => false, configurable: true});
// 验证: navigator.onLine === false ✅

// 步骤 2: 实际 fetch (真网络栈)
const r = await fetch("https://s.taobao.com/", {method: "GET", mode: "no-cors", cache: "no-store"});
// 结果: r.status === 200, duration === 121ms ✅
// 证明: 改 navigator.onLine 不影响真实网络栈, fetch 仍 200 OK 121ms
```

**结论**:
- **假断网**: 改 `navigator.onLine` 只改 JS 读数, 真实网络栈 / Service Worker / fetch **不受影响**
- **真断网需求**: 需 CDP `Network.emulateNetworkConditions({offline: true})` 真实模拟
- **真 gap**: 缺 `vortex_offline_mode` 工具 (CDP Network 域封装, 支持 offline / slow 3G / 慢速等)
- **附带误诊排除** (V2 评审 §1.4 订正): pass 判据"vortex_navigate 应有重试机制" — 自动重试会**掩盖真实错误**, 不应是真 gap, 移除
