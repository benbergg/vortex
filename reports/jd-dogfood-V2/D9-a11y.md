## 1. 维度概述

- 评测对象: ARIA 属性 (`aria-label` / `role` / `aria-labelledby`) / 焦点管理 (`activeElement` / Tab 焦点链) / 屏幕阅读器语义
- vortex 工具现状:
  - 缺专用 a11y 工具, 仅 `vortex_evaluate` 可读
  - `vortex_observe` 在 main frame 0 全部扫空 (V2 真发现, 见 §4.1)

## 2. 评测方法

- **主路径** (Opus 订正): `vortex_evaluate` 读 `aria-*` / `role` / `document.activeElement` (拆 `ariaLabelCoverage` vs `accessibleNameCoverage` 两指标, 不被文本兜底掩盖)
- **兜底**: `vortex_evaluate` 替代 `vortex_observe` 抓 a11y props (因 observe 不可用, 降级)
- **pass 判据**:
  - ✅ PASS: `ariaLabelCoverage` ≥ 90%
  - ⚠️ PARTIAL: 50-90%
  - ❌ FAIL: <50%
  - `accessibleNameCoverage` 作旁证记录
- **跨平台真品 id 来源**:
  - 京东 3 品类: V1 沿用 (3C=`100142621650` iPhone 16 128G 白色 / 家电=`100146042265` 海尔空调 / 服饰=`10163956330188` NAERSI 连衣裙本白色 L)
  - 淘宝 3 品类: V2 取搜索页第一个真品 (3C=`994000747172` iPhone 16 ProMax / 家电=`841228841152` 美的酷省电 tmall / 服饰=`902033968055` Anseola 女宝宝背心裙)
- **人化路径**: 京东因频控 (V2 后段被风控, 跳转到 `pc-frequent-pro.pf.jd.com?reason=403`) 后续评测**切到淘宝**; 跨品类走 `vortex_vortex_navigate` 到 s.taobao.com/search?q=<品类>, 商品详情用 `vortex_mouse_drag` 真实点击触发 SPA 跳转 (不直接 navigate 商品 URL, 避免频控)

## 6. 工具能力 gap 诊断

### 6.1 真 gap #1: vortex_observe 在 main frame 0 全部扫空 (原语层能力缺失, ROI 最高)

**现象**:
```
vortex_observe scope=viewport/full filter=interactive/all
  → "# frame 0 not scanned (url=...)"
```

**根因** (debug_read console error 实证):
```
ReferenceError: applyReactClickableMarker is not defined
    at <anonymous>:1:14814
    at <anonymous>:1:15441
```

**源码分析**:
- `packages/extension/src/handlers/observe.ts:228` `applyReactClickableMarker` 是 handler 顶层 export 函数
- 在 `scanOneFrame` (observe.ts:254) 用 `chrome.scripting.executeScript` 注入 page-side MAIN world 时, **`func` 参数被序列化为字符串** (background scope 中定义的 `applyReactClickableMarker` / `REACT_CLICKABLE_HINT` 不进字符串)
- dist build (`background.ts--x5mOpJ-.js`) 验证: 函数定义在 background scope, 引用在 inject func body 内, **序列化时引用对象未带过去**
- 结果: page-side MAIN world 抛 ReferenceError → `scanOneFrame` 返 null → "frame 0 not scanned"

**实测覆盖** (vortex_observe 在所有页面都不可用):
- search.jd.com ❌
- item.jd.com ❌
- s.taobao.com ❌ (淘宝同症状)
- example.com ❌ (验证非京东反爬 / X-Frame-Options 问题)
- 双平台一致 → **不是** 平台问题, **是** vortex 自身 bug

**影响**:
- 不只是 D9, 任何依赖 observe 的评测 (D11 BUG-012 回归 / D3 主路径交互) 都受影响
- V1 评测未发现, 因 V1 主路径是 extract + act + wait_for, observe 是 "兜底" 路径
- V2 D9 用 observe 抓 a11y props 直接踩雷
- V2 D11 BUG-012 回归也需降级 (走 fixture 或 mouse_drag, 不用 observe 拿 ref)

**修复建议** (V2.1 候选, 真原语层):
- **方案 A**: 在 page-side inject func 内**内联** `applyReactClickableMarker` 逻辑 (不依赖外部函数, **最简**, 0.3d)
- **方案 B**: 把 `applyReactClickableMarker` / `REACT_CLICKABLE_HINT` 移到 page-side 模块, 通过 `loadPageSideModule` 注入 (capture.ts 已有此模式)
- **方案 C**: 改 dist build (rollup config), 让 `applyReactClickableMarker` 真 inline 进 inject func

(具体 V2.1 P0 排序待 Phase 5 行动项填)

### 6.2 真 gap #2: 缺 vortex_a11y_audit 工具 (审计层空白, 定位待定)

**现象**:
- 京东**搜索/列表/详情** aria 4.76-13.92% / 淘宝 aria 0.97-9.79% — 双平台 a11y 普遍 < 15%
- 京东 60 个商品卡 div **0 个** 有 `role` / `aria-label`
- 缺一个工具能**扫整页 a11y 缺失** (类似 axe-core), 当前需手动 `vortex_evaluate` 逐个读

**建议**:
- 新增 `vortex_a11y_audit` 工具, 扫所有 `[role]` / `[aria-*]` 缺失, 输出 `axe-style` JSON 报告
- 注: 这是**审计套件层** (对标 axe-core), **非原语层** (对标 Stagehand/playwright-mcp). 是否做取决于 vortex 是否扩到审计层

## 7. 修复建议汇总

| 修复 | 优先级候选 | 工作量估 | 类型 |
|------|------------|----------|------|

## 8. 风险点 + 真站 vs fixture 边界

- **真站 vs fixture**: 京东淘宝真站 a11y 表现差, 不需 fixture 复现; 但 V2.1 修 observe 需用 fixture 验证 (`vortex-bench` 已有)
- **跨平台对比**: 京东 / 淘宝 a11y 表现均差 (京东详情略好, 淘宝搜索略好), 跨平台差异**非主要发现**, 主要发现是"电商 SPA a11y 系统性缺失"
- **诚实标注**: D9 18 格子 (3 品类 × 3 阶段 × 2 平台) 全 FAIL 是真结论, **不是** 评测方法问题
- **平台切换原因**: V2 后段京东风控被封 (`pc-frequent-pro.pf.jd.com?reason=403`), 后续评测切到淘宝 (V1 沿用平台, 缓存绕过率高)
- **路径选择原因**: 跨域直接 navigate 商品 URL 触发风控, 改用 `vortex_vortex_navigate` 到 s.taobao.com/search?q=<品类> + `vortex_mouse_drag` 真实点击商品卡, 走 SPA 站内路由 (带 referer + cookies, 模拟人)
