## 京东场景关键观察

### 1. **空名率与 3C/家电跨品类高度吻合** (31.5% vs 29.65% vs 31.0%, 差异 ±2pp 在合理范围)
- 服饰 31.5% 空名 link/button — 京东 DOM 结构性特征
- **根本原因**: 京东 logo 和客服图标(img/svg 渲染, accessibility tree 无 name)
- **跨品类印证**: 3 个品类(3C/家电/服饰)空名率均在 30% 左右, 是结构性事实, **不是 vortex 缺陷**

### 2. **D3 mouse_drag 1 步成功 (对比家电 3 步降级)**
| 品类 | D3 降级路径 | 结果 |
|------|-------------|------|
| 3C 阶段 2 | mouse_drag(1px) | ✅ 1 步成功跳转 |
| 家电阶段 2 | mouse_drag → onClick evaluate → navigate URL | ⚠️ 3 步降级 |
| **服饰阶段 2** | **mouse_drag(1px)** | **✅ 1 步成功跳转** |

- **关键发现**: 服饰 firstCard.onclick 接受真实 mouse 事件触发 (与 3C 一致), 家电不接受
- **推测原因**: 京东不同品类商品卡的 onClick handler 复杂度不同:
  - 3C / 服饰: handler 简单, mouse_drag 派发的 mousedown/mouseup 序列被 React 接收
  - 家电: handler 体内 `r.isAdv` + `r.advClientClickUrl` 链路复杂, React 事件流处理更严格
- **vortex 评测策略**: 跨品类 D3 应优先 mouse_drag (1px 抖动), 失败则降级到 navigate URL

### 3. **firstCardClass `_card_1fqso_83` 跨品类 100% 相同** (关键确认)
- 3C 阶段 2: `_card_1fqso_83`
- 家电阶段 2: `_card_1fqso_83`
- **服饰阶段 2: `_card_1fqso_83`**
- **结论**: 京东搜索页是**单一 React 应用跨品类复用**, 不是每个品类独立。CSS Module hash 完全相同证明同一 build 产物服务所有品类
