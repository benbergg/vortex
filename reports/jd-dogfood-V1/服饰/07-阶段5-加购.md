## 京东服饰场景关键观察(对比 3C / 家电 阶段 5)

### 1. **服饰颜色 SKU 切换 = 跨 SKU 整页跳转 (关键确认 - 与 3C/家电跨品类一致)**
- **3C 阶段 5 报告**: 京东价格不联动(走购物车合并结算)
- **家电阶段 5 实测**: 点击款式 SKU"小卧室推荐 大1匹" → URL `100146042265.html` → `100146042249.html` (整页跳转)
- **服饰阶段 5 实测**: 点击颜色 SKU"深青紫色" → URL `10163956330188.html` → `10163956330193.html` (整页跳转), 再点击尺码 SKU"XL" → `10163956330194.html` (再次整页跳转)
- **跨品类统一**: **京东详情页 SKU 切换 = 跨 SKU 整页跳转**(不是单页 AJAX 切换), 所有 state(标题/选中/库存)都重置
- **vortex 评测策略**: 跨品类 SKU 切换都需要"观察 URL 变化 + title 变化 + 商品名变化 + 选中状态保持"

### 2. **服饰跨 SKU 跳转后库存状态重新加载 (服饰独有的"颜色×尺码二维库存矩阵")**
- **本白色 L 状态**: M 无货 / L 选中 / XL / 2XL / 3XL 无货
- **深青紫色 XL 状态**: **M/L/XL/2XL/3XL 全有货**
- **核心发现**: 京东服饰**不同颜色对应不同尺码库存**, 跨 SKU 跳转后服务器重新返回库存
- **业务原因**: 服饰品牌按 "颜色 × 尺码 = SKU 编号" 管理库存, 比家电"系列+款式"复杂度高
- **vortex 评测策略**:
  - 服饰评测 D5 编程调用应在每次跨 SKU 跳转后**重新评估库存**(不能用初始状态)
  - 服饰评测 D3 切换 SKU 应**先评估库存避免 click 无货**(M/3XL 切换会 click 失效)

### 3. **服饰跨 SKU 价格保持 (高端品牌同价, 与家电跨 SKU 价格变化不同)**
- **家电跨 SKU 价格变化**: 大1.5匹 KFR-35GW ¥1699.15 → 大1匹 KFR-26GW ¥1656.65 (款式不同价格不同)
- **服饰跨 SKU 价格保持**: 本白色 L ¥1115 → 深青紫色 L ¥1115 → 深青紫色 XL ¥1115 (颜色/尺码不同同价)
- **业务原因**: 娜尔思 NAERSI 高端品牌"同款不同色/尺码"同价, 与家电"同系列不同款式"按规格定价不同
- **vortex 评测策略**: 跨品类 SKU 切换不一定带来价格变化, **不能用价格变化作为"SKU 切换成功"信号**, 应该用 URL 变化 + skuTitleName 变化

### 4. **react-virtuoso 虚拟列表 ELEMENT_OCCLUDED (跨品类一致, 与家电完全相同)**
- **3C 阶段 5**: 未触发 ELEMENT_OCCLUDED
- **家电阶段 5**: 多次触发 `<div._rateListContainer_1ygkr_45>` / `<div#rateList>` 遮挡
- **服饰阶段 5**: **1 次 `<div._rateListContainer_1ygkr_45>` 遮挡** (与家电同源 hash `1ygkr`)
- **关键确认**: react-virtuoso 虚拟列表 hash `1ygkr_45` 在家电/服饰跨品类完全一致 — 京东详情页 react 组件复用
- **vortex 跨品类评测策略**: 京东 react-virtuoso 列表元素 click 必须用 `evaluate scrollIntoView + click` 兜底

### 5. **服饰加购无附加服务勾选 (对比 3C AppleCare / 家电京选服务)**
- **3C 阶段 5**: AppleCare 4 档(¥299/¥649/¥1199/¥589), 用户主动勾选
- **家电阶段 5**: **京选服务默认勾选 10年免费换新 ¥94.90**, 用户主动取消
- **服饰阶段 5**: **无附加服务勾选** — 服饰加购流程更简单
- **业务原因**: 服饰是低客单价品类(¥1115 vs 家电 ¥1656.65 vs 3C ¥4172.51), 附加服务渗透率低
- **vortex 评测策略**: 服饰加购流程比 3C/家电简单, D4 维度直接跳过(无表单勾选)

### 6. **服饰加购按钮 class `"  "` (跨品类一致, 与 3C/家电完全相同)**
- 服饰加购按钮 `#add-to-cart` class `"  "` (两个空格, 装饰性)
- 与 3C/家电完全一致 — 京东 React 命名约定跨品类统一
- 加购按钮 tag DIV (非 button), 跨品类一致

### 7. **服饰加购成功验证三重奏 (跨品类一致)**
- **加购前**: cartCount=`6`(顶部"购物车 6")
- **加购后**: cartCount=`7`(+1 加购成功)
- **toast-box 触发**: 存在 `.toast-box` div (height=0, text="" - toast 已渲染但已淡出)
- **URL 不变**: 加购后仍停留在 `item.jd.com/10163956330194.html`, 京东不走"加购→购物车"自动跳转
- **vortex 加购验证三重奏**: 1) cartCount +1 2) toast-box 元素存在 3) URL 不变(停留详情页)
- **跨品类一致** (3C/家电/服饰): 京东加购流程统一

### 8. **服饰 SKU 切换 - 颜色后 + 尺码后双跳转 (服饰独有, 家电仅款式 1 次跳转)**
- **家电阶段 5**: 1 次跳转 (款式 SKU)
- **服饰阶段 5**: **2 次跳转** (颜色 SKU + 尺码 SKU)
- **vortex 评测策略**: 服饰评测应记录"颜色切换次数 + 尺码切换次数", 评估"完整购买流程的跨页跳转复杂度"

### 9. **直接 evaluate `getElementById('add-to-cart').click()` 兜底成功**
- 在 react-virtuoso ELEMENT_OCCLUDED 场景下, 直接 `evaluate document.getElementById('add-to-cart').click()` 比 `vortex_act click` 更稳定
- **避免 vortex_act 的 ELEMENT_OCCLUDED 检测**, 真实触发 React onClick handler 完成加购
- **vortex 跨品类评测经验**: 京东详情页加购按钮 click **优先用 evaluate 兜底**, 避免 occluded 检测

### 10. **服饰加购流程时间 vs 家电/3C 对比**
- **3C 加购流程**: click 加购 → cartCount +1 → toast (跨页 0 次)
- **家电加购流程**: click 款式 SKU → 跨页 1 次 → click 加购 → cartCount +1 → toast (跨页 1 次)
- **服饰加购流程**: click 颜色 SKU → 跨页 1 次 → click 尺码 SKU → 跨页 1 次 → click 加购 → cartCount +1 → toast (跨页 2 次)
- **vortex 评测价值**: 服饰加购流程跨页次数最多, vortex 跨页能力是服饰评测的关键
