## 京东场景关键观察

1. **空名率与 3C 阶段 2 高度吻合** (31.0% vs 29.65%, 差异 +1.35pp 在 2 个百分点内) — 京东 DOM 结构性特征而非偶发。**根本原因**(由 Phase 1 已确认): 京东 logo 和客服图标(img/svg 渲染, accessibility tree 无 name)。本次列表页 62/200 空名中, e10/e15/.../e75 的 link 大多对应客服图标, button 大多对应加购车按钮(图标无 name), textbox e74 是结果内"在结果中搜索"框(无 placeholder aria-label)。**意义**: 京东场景的 a11y 缺陷是稳定的、可预期的; vortex 在京东场景下, 大量"未命名 link/button"是结构性事实, **不应视为 vortex 缺陷**。

2. **D3 降级原因多层叠加** — 与 3C 阶段 2 D3 (mouse_drag 一次成功跳转) 不同, 本次家电 D3 需要 **三步降级**:
   - 步骤 1 (主路径): vortex_act click 暴露 ref → 客服 link/加购车 button, 不可用
   - 步骤 2 (降级 1): vortex_mouse_drag 真实 mouse 事件 → **未触发 firstCard.onclick 跳转** (与 3C 阶段 2 mouse_drag 一次成功不同!)
   - 步骤 3 (降级 2): vortex_evaluate 调 `__reactProps$.onClick` (handler 函数体执行了 st({spm:...})) → **URL 未变** (handler 内部需真实 React SyntheticEvent, 伪造 event 缺 nativeEvent.target.dataset 等)
   - 步骤 4 (降级 3): vortex_navigate 详情页 URL → 跳转成功
   
   **根本原因差异**: 3C 阶段 2 firstCard.onclick 接受真实 mouse 事件触发 (mouse_drag 1px 抖动即可), 本次家电 firstCard.onclick 似乎绑定了更复杂的 React 事件流 (handler 体内 `r.isAdv` + `r.advClientClickUrl` 链路), mouse_drag 派发的 mousedown/mouseup 序列未被 React 接收。**理论推测**: 京东新搜索页 React 重写后, firstCard 的 onClick 是 React 合成事件, mouse_drag 产生的 native MouseEvent 不会自动被 React 接收(需 react-dom 监听 root 事件再 dispatch); **vortex_mouse_drag 是否经过 CDP dispatchEvent 影响 React 事件接收不确定, 但本次确实未生效**。

3. **firstCard.onclick 桩函数 `function kd(){}`** + **hasReactProps: true** — 确认 React 18+ 合成事件。**意义**:
   - `element.click()` (编程派发) 不能触发 React 跳转
   - 真实 mouse 事件 (mouse_drag) 在 3C 阶段 2 能触发, 本次家电阶段 2 失效 — 京东搜索页 React 重写后事件流更复杂
   - 必须用 `vortex_navigate` 或 `vortex_evaluate` 调用 React 内部 router API(本次尝试调 `__reactProps$.onClick` 仍因 event 不完整失败)
   - **vortex_evaluate 触发 click 已验证无效** — 这是 React 框架通病, 非京东特有

4. **京东新搜索页 React 重写特征汇总** (与 3C 阶段 2 完全一致):

7. **D7 waitedMs=0** — 京东新搜索页 SPA 加载极快, 与 3C 阶段 2 一致 (3C 阶段 2 D7 waitedMs=1)。**家电搜索结果页比 3C 略快 1ms**, 可能是商品卡数量 (30 vs 29) 略多的边际差异。**D8 waitedMs=0** (navigate 后立即 ready) — vortex_navigate 性能极佳。

8. **wait_for IIFE 表达式解析问题** — 第一次 `vortex_wait_for(mode=custom, value="function(){...}()")` 报 Timeout(no response for page.waitForExpression after 20000ms), 推测是 vortex 把 value 当作 Playwright `page.waitForExpression` 表达式, 不识别 IIFE 包装。**改用裸表达式** `document.querySelectorAll(...).length >= 10 || ...` 后正常。**这与阶段 1 D7 失败是同一根因** (当时用了 `function(){return ...}` 也是 IIFE), 修正后两次都成功。

10. **D2 extract maxLength=5000 截断提示** — 列表页文本量 6354 字节, 5000 上限触发 VORTEX_TRUNCATED 警告 (3C 阶段 2 是 6646 字节, 家电略少)。**建议** vortex_extract 在京东列表页应支持 maxLength=8000 (给后续阶段兜底), 或调用时分批 extract (每段 maxLength=4000 滚动一次)。本次截断不影响关键字段召回(价格/评价/自营/家电特色标签 都在前 5000 字节)。
