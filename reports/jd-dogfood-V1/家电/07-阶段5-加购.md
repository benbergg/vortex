## 京东家电场景关键观察(对比 3C 阶段 5)

### 1. **款式 SKU 切换实际是跨 SKU 整页跳转(关键确认)**
- **3C 阶段 5 报告**: 京东价格不联动(走购物车合并结算)
- **空调阶段 5 实测**: 点击款式 SKU"小卧室推荐 大1匹" → URL `100146042265.html` → `100146042249.html` (整页跳转) + 商品标题从"大1.5匹 KFR-35GW" → "大1匹 KFR-26GW" + 价格 ¥1699.15 → ¥1656.65 + 累计评价 20万+ → 5万+
- **跨品类统一**: **京东详情页 SKU 切换 = 跨 SKU 整页跳转**(不是单页 AJAX 切换),所有 state(价格/标题/评价/SKU 选中/购物车)都重置
- **vortex 评测策略**: 京东 SKU 切换不需要"等价格联动",应观察 URL 变化 + title 变化 + 价格变化是否一致

### 2. **react-virtuoso 虚拟列表是京东详情页的隐藏陷阱(新发现)**
- **3C 阶段 5**: 没有触发 ELEMENT_OCCLUDED (用 div 模拟的 SKU 按钮直接 click 成功)
- **空调阶段 5**: 多次触发 ELEMENT_OCCLUDED,遮挡容器:
  - `<div._list_1ygkr_67>` (款式 SKU 列表)
  - `<div._rateListContainer_1ygkr_45>` (评价列表 rate list)
  - `<div#rateList>` (评价列表 id 选择器)
- **核心原因**: 京东详情页长列表(款式 SKU 10 个 / 评价 20万+)用 **react-virtuoso 虚拟滚动**,vortex click 检测到元素被 virtuoso 列表容器遮挡
- **vortex 跨品类评测策略**: 京东 react-virtuoso 列表元素 click 必须用 `evaluate scrollIntoView + click` 兜底,不能依赖 vortex_act click
- **.codegraph 启示**: `_list_1ygkr_67` / `_rateListContainer_1ygkr_45` 的 hash `1ygkr` 是同一个 react-virtuoso 组件实例(京东详情页 react 组件复用)

### 3. **京东家电延保服务 4 档 div 模拟 checkbox(对比 3C 优选服务)**
- **3C 优选服务**: AppleCare 4 档(¥299/¥649/¥1199/¥589), 维修类
- **空调京选服务**: 延保/换新 4 档(¥11.11/¥69.00/¥94.90/¥119.00), 整机延保类
- **class 结构**: 
  ```html
  <div class="service-item selected ">
    <span class="service-name">10年免费换新</span>
    <span class="service-price">94.90</span>
  </div>
  ```
- **class 末尾有空格**(`service-item  `, `service-item selected `) - 与 3C `#add-to-cart` class="  " 现象一致(京东 React 命名约定)
- **默认勾选状态**: 京东家电京选服务**默认勾选 ¥94.90 (10年免费换新)**,用户可手动取消(京东引导用户购买延保)
- **价格联动**: 京选服务价格不影响主商品价格(¥1656.65 不变),购物车合并结算时累加
- **vortex 评测策略**: 京选服务勾选用 `.service-item` div click,验证 `className.includes('selected')`

### 4. **选中 class 跨品类差异(BEM 缩写 vs 完整)**
- **3C 阶段 5**: 选中 class 是 `--sel`(BEM 缩写),如 `specification-item-sku--sel`
- **空调阶段 5**: 选中 class 是 `--selected`(完整),如 `specification-item-sku--selected`
- **重要发现**: 同一组件在跨 SKU 跳转前后 class 名可能变化!首次 observe 时是 `--selected`,跨 SKU 跳转后可能变为 `selected` 简化版
- **vortex 评测策略**: 选中检测应同时匹配 `[class*="--sel"], [class*="--selected"], [class*="selected"]` 三种模式

### 5. **加购成功验证(关键数据)**
- **加购前**: cartCount=`5`(顶部"购物车 5")
- **加购后**: cartCount=`6`(+1 加购成功)
- **toast-box 触发**: 存在 `.toast-box` div(虽然 height=0, text="" - toast 已渲染但已淡出)
- **URL 不变**: 加购后仍停留在 `item.jd.com/100146042249.html`,京东不走"加购→购物车"自动跳转
- **加购按钮 class**: "  "(两个空格,装饰性, 3C 报告一致)
- **vortex 加购验证三重奏**: 1) cartCount +1 2) toast-box 元素存在 3) URL 不变(停留详情页)

### 6. **空调价格不联动(确认 3C 报告)**
- **3C 阶段 5 报告**: 京东价格不联动,走购物车合并结算
- **空调阶段 5 实测**: 京选服务勾选前后主商品价 ¥1656.65 不变(延保在购物车合并)
- **款式 SKU 切换价格变化**: 不是"联动",是跨 SKU 整页跳转(URL 变化 + title 变化 + 评价数变化)
- **vortex 评测策略**: 京东价格"联动"实际是跨 SKU 跳转, 不是单页 AJAX 切换

### 7. **vortex click 降级策略: evaluate scrollIntoView + click(关键经验)**
- **触发条件**: 京东 react-virtuoso 虚拟列表 ELEMENT_OCCLUDED 错误
- **降级方案**:
  ```js
  // vortex_evaluate
  el.scrollIntoView({block: 'center'});
  el.click();
  ```
- **优势**: 绕过 vortex occluded 检测 + 真实触发 React onClick handler
- **3 阶段 + 5 阶段验证**: 多次成功(本次款式 SKU + 京选服务 + 加购按钮)
- **vortex 跨品类评测经验**: 京东详情页长列表元素 click **必须**用此降级方案,不能用 vortex_act click

### 8. **D7 wait_for custom 误命中(细节)**
- **第 1 次**: wait_for value=`!!document.querySelector('[class*="toast"]') || !!document.querySelector('[class*="cart-count"]')` → 1ms 命中(实际是 `#rateList` 元素被误匹配到 `[class*="toast"]`)
- **第 2 次**: wait_for value=无 `!!` 包裹 → `{ready: true, value: {}, waitedMs: 0}` 验证 toast-box 触发
- **降级方案**: wait_for value 用 `document.querySelector(...)` 直接返回元素(不用 `!!`),这样 value 是元素对象,可验证 className/textContent
- **vortex 评测经验**: wait_for custom value 应避免 `!!` 包裹,直接用元素对象更准确
