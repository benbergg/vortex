## 京东场景关键观察

1. **D3/D4 降级原因与 3C 阶段 1 不同** — 3C 阶段 1 的降级原因只有 D3 NOT_STABLE(顶部 sticky transition), fill 一次性成功。本次家电阶段 1 出现**两重新降级**:
   - **D4 fill NOT_STABLE + vortex_fill 不支持 force** — vortex_fill 在京东顶部 sticky 容器下会触发 NOT_STABLE(同 3C 阶段 1 D3 触发条件),且** vortex_fill 没有 force 兜底** (只有 vortex_act click 支持 force)。这是 vortex 自身的能力缺口。
   - **D3 click force=true 调用成功但状态未生效** — click 真实 mouse 事件确实派发了(CDP realMouse, x=958, y=74.5),但因 input.value 未同步到 React state.keyword, onClick handler 读 keyword 为空,未发起搜索 navigate。这是 React 受控组件 + 京东 SPA 路由的耦合问题。
   - **D3 vortex_evaluate 调 onClick handler 失败** — 找到 `__reactProps$.onClick` 直接调用,handler 函数体执行了(st({spm:...})),但因 SyntheticEvent 伪造不够完整(handler 内部可能用 nativeEvent.target.dataset), URL 未变。
   - **D3/D4 联合根因**: 京东搜索框是匿名 React 受控 input(无 id/name, 不在 form 内),`vortex_fill`/`vortex_act type`/`vortex_evaluate` 设值都不能真正同步到 React state。**唯一稳的方案是直接构造 search URL 用 vortex_navigate 跳转**。

2. **vortex_fill 缺 force 兜底** — 这是 Phase 4.1-4.7 第一次遇到的需求。3C 阶段 1 D4 fill 一次性成功,未暴露该缺口。本次家电阶段 1 因顶部 sticky transition + input 元素复合触发 NOT_STABLE,且** vortex_fill 的 options 不接受 force 参数**(只有 vortex_act click 接受)。**建议 vortex 后续优化**: vortex_fill 也应支持 force 兜底(参考 vortex_act click 的 force 语义)。**P1 修复路径**: vortex_fill 内部对 input.value 的 set + 派发事件时,应跳过 element stability 检查。

3. **京东搜索框结构特性**:
   - `<input type=text aria-label=搜索 id="" name="">` — 完全匿名
   - 不在 form 内 (没有 form 包裹)
   - 由 React 受控 (props.value = state.keyword, onChange = setKeyword)
   - 搜索按钮 `<button>搜索</button>` 在 React onClick 内调用 router.push(`/Search?keyword=${state.keyword}`)
   - **这种"匿名 React 受控 input + 无 form + 路由跳转"结构对自动化测试极不友好**。vortex_fill 设值走原生 input.value setter 路径,React 重写后 state 不变。**唯一稳的方案是直接构造 URL**。

4. **wait_for IIFE 表达式解析问题** — 第一次 `vortex_wait_for(mode=custom, value="function(){...}()")` 报 Timeout(no response for page.waitForExpression after 15000ms), 推测是 vortex 把 value 当作 Playwright `page.waitForExpression` 表达式, 不识别 IIFE 包装。**改用裸表达式** `location.href.includes('Search') || ...` 后正常。这与 3C 阶段 2 D7 (`document.querySelectorAll(...)`) 写法一致 — **wait_for value 必须是裸 JS 表达式, 不能是 function(){}() IIFE**。

5. **D5 hasKeyword URL encode 误判** — 第一次 evaluate `location.href.includes('空调')` 返回 false(URL 是 encoded `%E7%A9%BA%E8%B0%83`)。**修正**: 用 `decodeURIComponent(location.href).includes('空调')` 验证。后续 D5 评估应优先用 `pageKeyword = input.value` 或 `title.includes(keyword)` 验证,不依赖 URL。

6. **D6/D7 京东 SPA 跳转速度** — 1440x754 viewport 渲染 6 个空调商品卡(图片已加载),settle 600ms。**比 3C 阶段 1 慢 0.6s** (3C 阶段 1 D7 waitedMs=0) — 可能因空调商品卡图片资源比 3C 复杂(家电商品图更大)。**意义**: 家电搜索结果页 SPA 加载比 3C 略慢,但仍在 1s 内 ready。
