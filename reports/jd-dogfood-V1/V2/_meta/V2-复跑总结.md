- **真实空名率 0%**: 8e0dd9f 修复已正确处理京东商品卡 textContent 判定

### 2.2 P1-2 57fcbb0 修复泛化

**修复内容**: `packages/extension/src/action/auto-wait.ts:93-97`:
```ts
const lastReasonIsStability = lastReason === "NOT_STABLE";
throw vtxError(
  lastReasonIsStability ? VtxErrorCode.NOT_STABLE : VtxErrorCode.TIMEOUT,
```

## 3. V1 → V2 矛盾解释

V1 报告与 V2 实测的 3 处矛盾, 全部由"V1 时 vortex 进程加载旧 dist 缓存"解释:

| 矛盾 | V1 报告 | V2 实测 | 解释 |
|------|---------|---------|------|
| P1-1 京东空名率 29.90% | (旧口径 + 旧 dist) | 0% (新口径 + 新 dist) | 旧 dist 无 8e0dd9f 修复, 全链接空名率高; 新口径排除装饰性链接, 实际修复后 0% |
| P1-2 京东 errorCode=TIMEOUT | 旧 dist 518d500 之前, 无错误码修复 | NOT_STABLE (新 dist 57fcbb0) | 旧 dist 用 VortexErrorCode.TIMEOUT, 新 dist 改用 NOT_STABLE |
| P1-2 京东 hint 不含 force=true | 旧 dist 错误码路径错, 走 TIMEOUT hint | force: true 兜底建议 (新 dist 修复链) | 旧 dist 走 `errors.hints.ts` TIMEOUT hint, 新 dist 走 NOT_STABLE hint 含 force=true |
| P1-2 京东 message 格式 | "Actionability timeout after..." (旧分支) | "Element not stable after..." (新分支) | 旧 dist 走 else 分支, 新 dist 走 if 分支 |

**结论**: V1 评测**不反映修复后真实行为**, 因 vortex 进程加载旧 dist 缓存。V2 复跑确认 V4 修复在京东 100% 适用, V1 报告"修复未泛化"的幻影结论被推翻。
