## 京东 3C 加购阶段独有发现

### 1. 跨 SKU 跳转机制(同款不同颜色是不同 SKU)

| 颜色 | SKU 编号 | URL 路径 |
|------|---------|---------|
| 白色 | 100142621650 | `/100142621650.html` |
| 粉色 | 100142621614 | `/100142621614.html` |

- 点击颜色 → SPA 跳新 SKU URL(hash=`#switch-sku`)
- URL 变化但 DOM 大量复用(view 层不重建)
- 标题/榜单名/价格随 SKU 变化,但评价/SKU 树结构保持
- **NOT_ATTACHED 错误源于此**:上一 ref 所在页面被整体替换

### 2. 优选服务(AppleCare)不联动商品价

- 主商品价格 selector: `.product-price--value` = ¥4172.51
- 选延保后价格仍 ¥4172.51(预期 +¥299 = ¥4471.51,但未变)
- **京东设计:延保在购物车页合并结算**,详情页只标记已选服务
- class 切换: `service-item` → `service-item selected`(BEM modifier)

### 3. 优选服务非原生 checkbox

```javascript
// ❌ 找不到
input[type="checkbox"]  // 0 命中
// ✅ 实际是 div
.service-item           // 4 个 div
.service-item.selected  // 选中态
```

京东优选服务用纯 div 模拟 checkbox,无 form 元素。Vortex 需用 `div.service-item:has-text("...")` 文本定位。

### 4. #rateList 弹层残留(Phase 4.4 副作用)

- 上一阶段点击"全部评价"触发京东评价弹层 `#rateList`(z-index=1800,1440x754 全屏 fixed)
- **本阶段开始时弹层未自动关闭**
- 关闭按钮: `._closeIcon_1ygkr_39` (x=1157, y=75)
- **Phase 4.4 落档需加注**:评价弹层为 sticky full-screen,需主动关闭

### 5. 加购后 SPA 视图 rollback

- 加购成功后:URL 保持粉色 SKU,购物车数更新
- 但 SPA 视图自动回到默认"白色"页(可能是 JD 历史 bug 或快照)
- 实际数据:粉色 SKU 已加入,购物车数 5
- 建议后续阶段:加购后用 evaluate 验证购物车数,不要依赖截图

### 6. NOT_STABLE / OBSCURED 频发(同 Phase 4.4)

加购区位于 scrollY=295-340,该区间:
- 顶部 sticky 京东头部
- 浮动购物车栏
- 价格区 CSS transition

**所有 vortex_act 需 force=true 兜底**,5s timeout 后频繁 NOT_STABLE。

**关键 selector 沉淀**:

- `.service-item` / `.service-item.selected` 优选服务
- `.cart-count` 顶栏购物车数
- `#rateList` 评价全屏弹层
- `._closeIcon_1x0rp_261` 通用弹层关闭
- `100142621650.html`(白) / `100142621614.html`(粉) SKU 路径规律

**已知 quirk 沉淀**(供 Phase 5.x 复跑):
1. 加购前先关闭 `#rateList` 弹层
2. 颜色切换触发跨 SKU 跳转 → ref 失效需重 observe
3. 优选服务 class 是 modifier 风格,选中加 "selected"
4. 加购后 SPA 自动 rollback 视图,evaluate 验证购物车数为准
5. 延保不联动商品价,需购物车页验证总金额
