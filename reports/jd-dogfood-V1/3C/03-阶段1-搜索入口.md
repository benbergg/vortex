## 京东场景关键观察

1. **NOT_STABLE 触发原因** — 京东顶部搜索栏在 sticky/fixed 容器中,有 CSS transition(疑似 `transition: bottom 0.15s` 或类似 hover/active 动画)。首次 click 必触发 NOT_STABLE,`force: true` 100% 兜底通过。**这与 Phase 2 详情页 NOT_STABLE 触发条件不同**(详情页是价格区 transition,搜索栏是顶部 transition),但兜底方案一致。

2. **D1 observe 一次抓全** — 与 Phase 1 P1-1 复跑发现的"京东空名率 29.90%"形成对比:**本次搜索框/按钮 ref 名都带语义**("搜索" textbox/button),没有"未命名 div"问题。原因是顶部搜索栏是京东长期稳定的 layout,DOM 命名规范;问题主要出在侧边客服图标(京东 logo/客服图标)——**本次 3C 路径不涉及侧边栏**。

3. **D5 productCount selector 失配** — `li.gl-item` 是京东老搜索页 class(2018 前后),新搜索结果页已改用 React 重写,class 名动态化。这是**辅助验证口径问题**,不是 vortex 能力缺陷。后续阶段(02-商品列表)需要用 observe 重新发现 selector。

4. **D4 fill 一次性成功** — 京东搜索框对 `vortex_fill`(`element.value` + 事件)响应良好,无需 `vortex_act` `type` 兜底(淘宝/天猫历史上 type 模式更稳)。**这与淘宝的"type 优于 fill"经验不同**——京东 SPA 对 fill 友好。

5. **D7 wait_for waitedMs=0** — force=true click 后 navigate 几乎同步完成(京东 SSR 跳转很快),无需额外 idle 等待。这是京东 vs 淘宝的关键差异:淘宝搜索跳转常需要 `wait_for idle dom` 兜底,京东直接 ready。
