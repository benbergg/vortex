## 京东场景关键观察

1. **Phase 4.2 遗留问题解决:商品标题 selector** — Phase 4.2 D8 报告 `.product-title`/`[class*="product-intro"] h1`/`[class*="sku-name"]` 三个候选全部 null。本次探查发现:
   - 文档唯一 h1 是"最小单价计算器"浮层,**详情页商品标题不是 h1**
   - 真正的标题 selector 是 `span.sku-title-name`(BEM 风格,**无 hash,稳定**)
   - 京东详情页 selector 比列表页稳定 —— 列表页用 React CSS Module `_card_<hash>_<line>`,详情页用 BEM 风格 `sku-title-name` / `page-right-skuname`
   - **vortex 在京东详情页的 selector 策略**:BEM 风格 + 父级 fallback(`div.sku-title` / `div.page-right-skuname`)

2. **Phase 0 描述的旧京东详情页 selector 全部失效**:
   - `.Ptable` / `.Ptable-item` / `dl` / `dt` / `dd` —— 京东新详情页规格表已重写为 `.attrs` / `.item ` / `.label` / `.value` 命名空间
   - `.parameter2` —— 不存在;`.parameter` 实际是**图片轮播组件的"规格参数"图项**(`image-carousel-track` 内),不是规格表
   - **`#add-to-cart` id 仍然存在**(Phase 0 描述正确),是 div 不是 button,class 是两个空格(装饰性,无意义)

3. **京东"查看全部参数"是页内锚点跳转,不是浮层** — force=true click 后:
   - URL 不变(`item.jd.com/100142621650.html`)
   - scrollY 从 40 → 2647(滚动到页面中部)
   - 无弹层蒙层(`hasMask: false`)
   - 京东用"scroll-to-section"代替"模态弹窗"展示规格,这是 SPA 性能优化(避免重 DOM 创建/卸载)。**vortex 用户需知**:在京东详情页点"查看全部参数"应观察 `scrollY` 变化而非 `hasMask` 变化

4. **NOT_STABLE 触发原因确认是顶部 tab sticky CSS transition** — Phase 4.1 搜索栏触发、Phase 4.2 列表页没有触发,本次详情页 `@2333:e45` 触发。**京东 SPA 中任何"sticky + CSS transition" 元素首次 click 必 NOT_STABLE**,`force: true` 100% 兜底通过。**京东 SPA 的 NOT_STABLE 触发条件可总结**:scrollY 接近 0 + 元素位于 sticky/fixed 容器内 + 有 CSS transition

5. **京东详情页加购按钮是 `<div id="add-to-cart">`** — **Phase 0 描述正确**,可直接用 id 选择器;但 observe 抓出的是 div 标签(@2333:e71,e72 都是 div),不是 button。**vortex_observe 揭示的事实**:京东详情页底部"加入购物车/立即购买"在 a11y tree 中是 div(无 role),真实 button 状态由 React 接管。后续 06 阶段(加购)用 `#add-to-cart`(稳定),Phase 2 已确认需 force=true 兜底

6. **京东详情页元素密度(73)远低于列表页(199)** — 详情页 73 元素中包含 14 个空名(顶部/底部导航)+ 6 个 span 规格版本/颜色 + 6 个 div 浮动栏 + 加购按钮等。**空名率**:14/73 ≈ 19.2%,低于 Phase 4.2 列表页 29.65%(详情页结构化更好,命名规范)

7. **D5 规格表 selector 反向工程**:
   - 第一次用 `attrs.querySelectorAll('.item')` 返回 0(看似失败,实际是 class 末尾带空格 `'item '` 导致)
   - 改用 `:scope > [class*="item"]` 才命中 29 项
   - **vortex 编程调用在京东 DOM 的陷阱**:`querySelectorAll('.item')` 不等于 `querySelectorAll('[class*="item"]')`,**京东 React 重写时 class 末尾会带空格作为 hash 边界标志**,严格 class 匹配会失效
   - **经验法则**:在京东新 SPA 中,**优先用 `[class*="xxx"]` 或 `:scope >` 限定,而非 `.className`**(避免末尾空格陷阱)

8. **京东详情页 tab 列表与淘宝/天猫根本不同**:
   - 淘宝详情页 tab:商品详情 / 规格参数 / 商品评价 / 售后服务
   - 京东详情页 tab:大家评 / 店铺 / **商品详情** / 售后保障 / 推荐(无独立"规格参数" tab,规格通过"查看全部参数"页内跳转)
   - 京东 tab 区只有 5 个,无 N 维评价标签。**京东评测场景中,规格数据要主动滚动到规格区获取,不能在 tab 上点击**
