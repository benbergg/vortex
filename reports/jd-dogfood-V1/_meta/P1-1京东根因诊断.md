### 3.2 京东空名 `<a>` 的实际判定路径

**客服图标 `<a class="_newIcon_zclqt_32 _customer_service_icon_zclqt_60">`(30 个,16x16)**:
```js
{
  childNodes: [],         // 无
  children: [],            // 无子元素
  textContent: "",         // 空
  title: null,             // 无 title
  querySelector('a[href],button,...'): null  // 无交互后代
}
```

判定流程:
1. `hasDirectText = false`(`childNodes` 全是 element node,无 TEXT_NODE)
2. 跳过 `if (hasDirectText) return ...`
3. `isContainer = false`(无子交互元素)
4. `text = normName("") = ""` → 跳过 `if (text && !isContainer) return text`
5. `titleAttr = null` → 跳过 title 兜底
6. `isContainer = false` → 跳过 `if (isContainer) return ""`
7. `iconNameFromClass` 走 className hash,但类名 `_newIcon_zclqt_32 _customer_service_icon_zclqt_60` 不匹配 iconFontName 前缀
8. `controlRoleFromClass` 不命中
9. `iconFontName` 不命中(无 `bi-/fa-/glyphicon-/vxe-icon-/van-icon-` 前缀)
10. **返 `""`(空名)**

**京东 logo `<a href="https://www.jd.com/">`(1 个,190x58,纯背景图)**:
同上路径,返 `""`(空名)。

## 4. 根因

### 4.1 京东空名 link 的本质

京东搜索结果页有 **31 个空名 `<a>`**(30 个 chat.jd.com 客服图标 + 1 个京东 logo),**全部是图标式 link**:
- `directTextNodes = []`
- `directElementChildren = []`
- `textContent = ""`
- `childCount = 0`
- **没有可读信息**(href 含 URL-encoded 商品 ID,但 observe 不读 href)

### 4.2 京东与淘宝根因对比 — **2026-06-08 审核后修正**

| 维度 | 淘宝 (V4 失败) | 京东 (V1 失败) |
|------|----------------|----------------|
| 空名 `<a>` 实体 | 整张商品卡 `<a class="doubleCardWrapperAdapt">` | 客服图标 `<a class="_newIcon_zclqt_32 _customer_service_icon_zclqt_60">` + 京东 logo `<a href="https://www.jd.com/">` |
| 尺寸 | 235×374 (整张卡) | 16×16 / 190×58 (图标 / logo) |
| `textContent` | 含 `¥88.23 标题 销量 回头客3万`(完整商品信息) | `""`(空字符串) |
| `directTextNodes` | `[]` | `[]` |
| 失败机制 | 8e0dd9f `PRODUCT_HINTS` 判定不命中(`¥/人付款/回头客/已售/月销` 都缺) + `isContainer=true`(内嵌店铺子 a)→ 返空名,**信息丢失** | 8e0dd9f `PRODUCT_HINTS` 判定不命中(空字符串无商品特征) + `isContainer=false` → 走 fallback 返空名,**本来就无信息** |
| 8e0dd9f 修复能否解决 | ❌ 待 V2 实测(`PRODUCT_HINTS` 关键词是否覆盖淘宝商品卡 textContent) | N/A (商品卡不是 `<a>`,无 textContent 信息) |
| 修复方向 | 待 V2 实测 8e0dd9f 泛化结果 | 给图标式 link 合成 `icon-link @x,y` 兜底名 或 在 observe 阶段过滤 chat.jd.com 链接 |

### 4.3 关键洞察

**京东与淘宝空名率数字都 ~30% 是巧合,根因完全不同**:

- **淘宝**:空名 link 有信息但没抓 → 需要**重新提取** (8e0dd9f 修复方向, 待 V2 实测)
- **京东**:空名 link 无信息 → 需要**合成兜底名** 或 **识别后过滤**

8e0dd9f P1-1 重做修复对**淘宝待 V2 实测**(`textContent` 含信息但修复用 `PRODUCT_HINTS` 关键词覆盖度待确认),对**京东无关**(商品卡不是 `<a>`,无 textContent 信息)。**审核后 BUG-009 重定性为 REQ-009-京东客服图标降噪**(原 P1 → P2)。

## 5. Patch 草稿(京东场景)

### 5.1 方案 A:图标式 link 合成 `icon-link @x,y`(推荐)

```ts
// packages/extension/src/handlers/observe.ts, 在 hasDirectText 判定后、isContainer 前插入
const bbox = el.getBoundingClientRect();
const isIconLink =
  el.children.length === 0 &&
  !el.textContent.trim() &&
  (bbox.width <= 32 || bbox.height <= 32) &&
  /^(https?:)?\/\//.test(el.getAttribute("href") || "");
if (isIconLink) {
  return `icon-link @x=${Math.round(bbox.x)},y=${Math.round(bbox.y)}`;
}
```

**优势**:
- 跨平台通用(任何 ≤32px 无文本 `<a>` 都给固定名)
- 立即降京东空名率 29.90% → ~1%(仅剩 190×58 logo 一个空名)
- 风险:32×32 图标按钮(如购物车图标)可能也被改 → 需后续细化

**劣势**:
- 信息密度低(LLM 仍要 click 才知道是客服)

### 5.2 方案 B:识别 chat.jd.com 客服链接给固定名(京东定向)

```ts
const href = el.getAttribute("href") || "";
if (/chat\.jd\.com|webchat\./.test(href) && !el.textContent.trim()) {
  return `chat-link @x=${Math.round(bbox.x)},y=${Math.round(bbox.y)}`;
}
```

**优势**:
- 简单直接
- 立即降京东空名率 29.90% → ~1%

**劣势**:
- 写死京东域名,跨平台扩展性差
- 若客服浮窗本身需被 observe 看到,需额外处理

### 5.3 方案 C:在 observe 输出阶段过滤 chat.jd.com(影响最小)

```ts
// 跳过 emit, 不进 observe 数组
if (/chat\.jd\.com|webchat\./.test(el.getAttribute("href") || "")) {
  return;  // 跳过 emit
}
```

**优势**:
- 不增加 observe 元素数,降噪
- 修复后空名率 29.90% → ~1%

**劣势**:
- 客服入口不可见,若 task 想"打开客服"需另行引入
- 平台特定(类似方案 B)

### 5.4 推荐方案 A + B 组合

先方案 A(通用图标 link 兜底),再方案 B(京东 chat 链接优先),两者叠加:
- 京东:32 个空名 → 0 个空名 + 32 个有意义的 `icon-link @x,y` / `chat-link @x,y`
- 淘宝:d4b7330 + 方案 A:29.9% → ~1%(淘宝商品卡是 `<a>` 但 directTextNodes=[] 仍空,方案 A 给固定名)
- 抖音 / 小红书:同样受惠(若它们也有客服图标式 link)

## 6. 风险点

| # | 风险 | 缓解 |
|---|------|------|
| 1 | 方案 A 误判 32×32 真按钮 link(如购物车图标) | 加白名单(类名匹配 `cart-icon / shopping-cart / kefu-icon`)或尺寸上调到 24×24 |
| 2 | 方案 B 写死京东域名,京东重命名 chat 子域后失效 | 改用 regex 模式 (`chat.*jd.com` / `kefu.*jd.com`) 容忍子域重命名 |
| 3 | 方案 C 客服入口不可见,影响"打开客服"任务 | 改用方案 A+B(给固定名而非过滤),客服浮窗 click 仍可工作(observe ref 还在) |
| 4 | 京东详情页客服浮窗与列表页同 className | 浮窗中客服按钮 bbox > 32px,方案 A 不命中,行为不变 |
| 5 | 淘宝 BUG 仍未修(本 patch 不解决淘宝"整张卡是 a"问题) | 淘宝需独立 patch(沿用 N0059-V4 §7.4 推荐方案 A:判 `textContent` 含商品特征) |
| 6 | 30 个 `chat-link @x,y` 占 observe 输出,信息密度下降 | 接受 trade-off(从空名 → 固定名,总比无信息好);后续可优化成 `chat-link (to 欣妃羽服饰京东自营旗舰店)` |
| 7 | 现有 unit test `extension 793` 全量无回归需重新跑 | 跑 `pnpm test` 验证 |
