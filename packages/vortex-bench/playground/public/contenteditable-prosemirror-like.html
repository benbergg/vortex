contentEditable + ProseMirror-like input pipeline

The top editor mirrors a real rich-text editor (ProseMirror, Slate, Lexical): it preventDefault's beforeinput, re-renders its DOM content from an internal model, and only accepts isTrusted=true (native browser-source) input events. Synthetic InputEvent dispatches are dropped. The bottom editor is a plain contentEditable div with no input interception, but it has the same vortex_fill / vortex_type entry-point problem: el.value doesn't exist on a div, so the host-side input-value-setter path throws Illegal invocation and the keyboard-event path's if (el.value !== undefined) guard silently no-ops.
